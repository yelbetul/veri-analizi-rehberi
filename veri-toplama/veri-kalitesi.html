Doğrulama Motoru

Esnek ve güçlü veri doğrulama kuralları ile veri bütünlüğünü sağlayın

🚀 Ne İşe Yarar?

Veri doğrulama İş kuralı uygulama Şema kontrolü Çapraz referans

Kullanım Amacı: Verilerin önceden tanımlanmış şema, iş kuralları ve referans bütünlüğü kurallarına uygun olup olmadığını kontrol etmek. Örnek: Müşteri verileri yüklenirken e-posta formatı, yaş aralığı ve benzersiz müşteri ID'si kontrolleri yapmak.

Şema Doğrulama (JSON Schema, XML XSD)

💡 Ne İşe Yarar?

Verinin yapısal olarak tanımlanan şemaya uygun olup olmadığını kontrol eder

  • • JSON Schema: JSON verilerinin yapısını ve veri tiplerini doğrular
  • • XML XSD: XML belgelerinin yapısal bütünlüğünü kontrol eder
  • • Avantajlar: Standartlaştırılmış, yeniden kullanılabilir, otomasyona uygun
  • • Kullanım: API yanıtları, veri değişim formatları, konfigürasyon dosyaları

İş Kuralı Doğrulama Çerçevesi

💡 Ne İşe Yarar?

İş mantığına özgü karmaşık kuralları uygular ve doğrular

  • • Koşullu Kurallar: "Eğer X ise, Y doğru olmalı" tipi kurallar
  • • Hesaplanmış Değerler: Formül tabanlı doğrulamalar
  • • İş Akışı Kuralları: Süreç bazlı doğrulamalar
  • • Örnek: "Kredi başvurusu onaylanmışsa, müşteri yaşı 18+'dan büyük olmalı"
Örnek JSON Şema Doğrulama Kuralı
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "customerId": {
      "type": "string",
      "pattern": "^CUST-[0-9]{6}$"
    },
    "email": {
      "type": "string",
      "format": "email"
    },
    "age": {
      "type": "integer",
      "minimum": 18,
      "maximum": 100
    },
    "orderAmount": {
      "type": "number",
      "minimum": 0
    }
  },
  "required": ["customerId", "email", "age"],
  "additionalProperties": false
}

15+ Veri Kalite Boyutu

Veri kalitesini kapsamlı bir şekilde değerlendirmek için çok boyutlu yaklaşım

🚀 Ne İşe Yarar?

Kapsamlı değerlendirme Çok boyutlu analiz Ölçülebilir metrikler İyileştirme odaklı

Kullanım Amacı: Veri kalitesini tek bir metrik yerine çok boyutlu bir yaklaşımla değerlendirmek, iyileştirme için öncelikli alanları belirlemek ve kaliteyi ölçülebilir hale getirmek. Örnek: Müşteri veri tabanının doğruluk, tamlık, tutarlılık, zamanında olma gibi 15 farklı boyutta analiz edilmesi.

Temel Kalite Boyutları

Doğruluk (Accuracy)

Verinin gerçek dünyayı doğru temsil etme derecesi

Ölçüm: Doğru kaynaklarla karşılaştırma, doğrulama testleri

Tamlık (Completeness)

Gerekli tüm verilerin mevcut olma derecesi

Ölçüm: Boş/null değer analizi, zorunlu alan kontrolü

Tutarlılık (Consistency)

Veriler arasında çelişki olmama derecesi

Ölçüm: Çapraz tablo kontrolü, iş kuralı uyumluluğu

Zamanında Olma (Timeliness)

Verinin ihtiyaç duyulduğunda hazır olma derecesi

Ölçüm: Güncelleme sıklığı, veri tazeliği metrikleri
