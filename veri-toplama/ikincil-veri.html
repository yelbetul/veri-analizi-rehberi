API Entegrasyonları

API'lerden veri çekme, rate limit yönetimi ve optimizasyon stratejileri

🚀 Ne İşe Yarar?

Gerçek zamanlı veri Üçüncü parti entegrasyon Otomatik veri güncelleme Servis mimarisi

Kullanım Amacı: Harici servislerden güncel verileri otomatik olarak çekmek ve sisteminize entegre etmek. Örnek: Hava durumu API'sinden veri çekip lojistik planlaması yapmak.

💡 Ne Zaman Önemli?

Büyük veri setleri çekerken, yüksek frekanslı isteklerde veya ücretli API'ler kullanırken rate limit yönetimi kritik öneme sahiptir.

Rate Limit Türleri
  • Per Second Limits
    • 10 requests/second
    • 1000 requests/minute
  • Daily/Weekly Limits
    • 10,000 requests/day
    • 50,000 requests/week
  • Burst Limits
    • Anlık yüksek trafik
    • Kısa süreli limit artışı
Yönetim Stratejileri
Retry Mekanizmaları

• Exponential backoff
• Jitter ekleme
• Circuit breaker

Önbellekleme

• Response caching
• Local storage
• CDN kullanımı

Python ile Rate Limit Yönetimi
import time
import requests
from requests.adapters import HTTPAdapter
from requests.packages.urllib3.util.retry import Retry

# Retry stratejisi
retry_strategy = Retry(
  total=3,
  backoff_factor=1,
  status_forcelist=[429, 500, 502, 503, 504],
)

# Session oluşturma
session = requests.Session()
session.mount("https://", HTTPAdapter(max_retries=retry_strategy))

# Rate limit kontrolü
def make_request_with_backoff(url):
  for attempt in range(5):
    response = session.get(url)
    if response.status_code != 429:
      return response
    time.sleep(2 ** attempt) # Exponential backoff
  return None

# Kullanım: Rate limit'e takılmadan güvenli istek
response = make_request_with_backoff('https://api.example.com/data')
if response:
  print('Veri başarıyla alındı')
else:
  print('İstek başarısız oldu')
